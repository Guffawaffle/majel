## Shipped Since Last Sweep

### Admin Panel Email Removal — Data Minimization (GDPR Article 5(1)(c))
- 6 admiral endpoints switched from `email` to `userId` as identifier
- `GET /api/auth/admiral/users` strips email from response via destructuring
- `AdminUser` frontend type drops `email` field
- `AdmiralView.svelte` email column removed; action handlers take `(userId, displayName)` instead of `(email)`
- Confirm dialogs show `displayName` instead of email
- `isSelf` check uses `u.id === currentUserId` instead of email comparison
- UUID validation (`UUID_RE`) added to all 5 admin action endpoints

### Audit Log PII Scrub
- Removed email from `detail` JSONB in 6 audit events: `auth.signup` (success + failure), `auth.signin.success`, `auth.signin.failure`, `auth.resend_verification`, `auth.password.reset_request`
- User identity remains captured via `actorId` UUID — no forensic capability lost
- `auth.unverified_cleanup` GC event no longer logs purged emails
- `cleanupUnverifiedUsers()` returns user IDs instead of emails; SQL `RETURNING` clause updated

### Right-to-Erasure — User Deletion Cascade
- `deleteUser()` now runs transactional cleanup across 14 tables before deleting the user record
- Crew tables: `officer_reservations`, `plan_items`, `fleet_presets`, `docks`, `loadouts`, `below_deck_policies`, `bridge_cores` (internal FKs cascade children)
- Chat: `chat_runs`, `operation_events`, `operation_streams`
- Settings: `user_settings`
- FK-cascaded (unchanged): `user_sessions`, `email_tokens`
- Uses `SAVEPOINT` per table so missing schemas in test DBs don't abort the transaction

### Audit Log Retention Policy (90-day)
- Append-only trigger updated: UPDATE still blocked; DELETE allowed only when session variable `audit.retention_purge = 'true'` is set
- `purgeOlderThan(interval)` added to `AuditStore` interface and implementation using `SET LOCAL` for transaction-scoped authorization
- Wired into hourly GC cycle — purges entries older than 90 days

### Audit Log API Sanitization
- `GET /api/admiral/audit-log` now strips `ipAddress`, `userAgent`, and legacy `email` from detail before returning to admins
- Prevents PII exposure even from historical audit entries written before the scrub

### Production Startup Guard
- `resolveConfig()` throws if `NODE_ENV=production` and `DATABASE_URL` is not set
- Prevents fallback to hardcoded `postgres://majel:majel@localhost:5432/majel` dev credentials in production

- ✅ Pino logger auto-redacts tokens, passwords, secrets, auth headers, cookies
- ✅ Error handler sanitizes internal details from client responses
